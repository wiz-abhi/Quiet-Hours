# Submission Checklist

Mirrors the Devpost requirements for the **Slack Agent Builder Challenge**. Work top to bottom; nothing here is optional for submission.

## Track & description
- [ ] Track selected: **Agent for Good**
- [ ] Text description written, including an explicit **social-impact section** (on-call / volunteer burnout — the human, not just the service)
- [ ] Description names all three required technologies and where each is used (RTS · MCP · Slack AI)

## Demo video (≤ 3 minutes, public)
- [ ] Video is **public** (YouTube/Vimeo, not private/unlisted-only if the rules require public)
- [ ] Under **3 minutes**
- [ ] **Slack RTS** visible on screen (detection firing — the 4-signal trigger)
- [ ] **MCP** visible on screen (`get_oncall` / `page_backup` via the PagerDuty MCP server)
- [ ] **Slack AI** visible on screen (intervention DM / AI-drafted handoff / morning Canvas)
- [ ] Video link added to the Devpost submission

## Architecture & repo
- [ ] **Architecture diagram uploaded** to the submission (from `docs/ARCHITECTURE.md`)
- [ ] **Public GitHub repo** with the full source and README
- [ ] README explains setup and the three required technologies

## Sandbox & judge access
- [ ] Slack **sandbox URL** included in the submission
- [ ] `slackhack@salesforce.com` invited to the sandbox
- [ ] `testing@devpost.com` invited to the sandbox
- [ ] Both judges confirmed able to access the workspace and the Quiet Hours app

## Deployment uptime
- [ ] **Deployed instance stays up July 14 – Aug 6** (judging window) — the app must be reachable and functional, not just the repo

## Submit
- [ ] Everything above complete
- [ ] **Submitted on Devpost by July 12** (ahead of the July 13 deadline)
